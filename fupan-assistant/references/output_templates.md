# 输出模板

## 复盘卡模板

对话结束时输出。用 Markdown 格式：

```markdown
## 复盘卡：{一句话事件标签}

**复盘对象**
- 事件：{一句话描述}
- 类型：{个人复盘 / 小团队复盘 / 项目复盘 / AI 协作复盘}
- 日期：{事件日期}

**预期—结果对比**
- 当时预期：{目标/目的/假设}
- 实际结果：{关键事实、数据、过程}
- 差异：{预期 vs 结果的 gap}

**复盘深度**
- 本次最深挖到：L{1-5}
- L1 主观感受：{感受方向 + 面向未来表达}
- L2 客观事实：{列出的事实；如未通过，标注「缺事实」}
- L3 背后原因：{根因链条；如未通过，标注障碍类型}
- L4 底层模型：{提炼出的规律/模型/框架；如未通过，标注「待深挖」}
- L5 能力建设：{识别出的能力短板；如未触及，标注「本次未触及」}

**下次行动**
- 失败复盘 — 改进：{3-5 条具体改进动作}
- 成功复盘 — 固化：{3-5 条应该保留和固化的做法}
- 九宗罪自检：{本次踩了哪些坑、做对了哪些}
- 如是团队 lead：团队复盘所处阶段（四步法）

**Wiki 标记**
- 状态：{待深挖 / 已沉淀}
- 目标域：{由复盘对象判断的域}
- 摘要页：`摘要-复盘-{一句话事件标签}-{YYMMDD}.md`
```

## Wiki 待深挖标记

复盘深度未达到 L4，或原则自检亮起信号时，自动输出：

```markdown
## 待深挖：{一句话事件标签}

**当前状态**
- 深度：L{1-4}（L5 为终点，不再标记待深挖）
- 已有结论：{一句话}

**卡在哪**
- 障碍：{缺事实 / 缺案例 / 缺积累 / 缺外部视角}
- 原则自检：{没区分事实和感受？没明确预期？视角单一？}

**什么时候回来**
- 触发条件：{具体条件，如「Q3 数据出来」「再积累 3 个类似案例」}

**建议处理**
- 在目标域 `log.md` 中记录待深挖项，等待触发条件满足后再归档。
- 暂不创建概念页，只写入摘要页（标记为待深挖）。
```

## Wiki 已沉淀标记

复盘深度达到 L4（产出至少一条可复用规律），自动输出：

```markdown
## 已沉淀：{一句话事件标签}

**沉淀内容**
- 模型/规律：{一句话}
- 可复用场景：{什么情况下可以直接用}

**下次激活**
- 当遇到 {类似事件类型} 时，调用此模型

**建议入 Wiki**
- 摘要页：`摘要-复盘-{一句话事件标签}-{YYMMDD}.md`
- 概念页：[[{模型/规律名}]]
- 相关概念：[[{相关概念1}]]、[[{相关概念2}]]
```

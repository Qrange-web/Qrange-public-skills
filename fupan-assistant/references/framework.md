# 复盘核心方法

> **框架来源声明**：本文档中的复盘框架（五层冰山、四象限、九宗罪等）来自资深实践者从几百次复盘经验中归纳的方法论，属于实践者经验框架（E3-E4），非实证研究模型。五层切割和判断标准逻辑自洽且有内部案例支撑，但「深度=价值」是隐含假设，非已验证的因果规律。Skill 使用时不应以「科学模型」的口吻引用。

## 一、复盘三要素

复盘本质上是一种学习的手段——基于过去真实经验，面向未来工作的一种学习。

| 要素 | 含义 | 反面 |
|------|------|------|
| 真实经验 | 你自己过去做的事 | 不是书上的、听来的、假设的 |
| 面向未来 | 为下一次做类似事情做准备 | 总结过去本身不是目的 |
| 学习 | 让自己变强的手段 | 不是甩锅、夸夸、吐槽、应付 |

三个要素都有就是复盘，缺一个就不是。

## 二、五层冰山模型

复盘深度从浅到深分五层。每多挖一层，价值大一层。

| 层级 | 名称 | 核心动作 | 判断标准 |
|------|------|---------|---------|
| L1 | 主观感受 | 说出「这次感觉好/不好」 | 能说出明确的感受方向 |
| L2 | 客观事实 | 还原事件过程，列出事实 | 至少 2 个具体事实（时间/地点/数字/动作/原话，有数据更好） |
| L3 | 背后原因 | 追问「为什么会这样」，至少两层 | 根因链条深挖两层以上，成功复盘需区分「刻意做到」vs「可能运气」 |
| L4 | 底层模型 | 提炼可复用的规律/框架/清单 | 产出至少一条可复用的规律 |
| L5 | 能力建设 | 挖到团队/自己的能力短板 | 能说出一个能力短板，且能解释补齐后其他事也会改善 |

**L1 操作技巧**：引导面向未来表达——「下次我会…」而不是「这次太…」。

**L3 辅助工具**：转化率问题→动力阻力模型；扩张亏损→单元模型；目标未完成→项目管理框架。

**L4 常见沉淀**：关键成功要素（3-5 个）、雷达图筛选模型、N 步法操作流程、执行清单。

**L5 典型信号**：问题本质不是事情本身，而是团队/组织的基本功不足。补齐一个能力短板，不止这一个问题改善。

## 三、四象限定位

复盘前先定位，不要用执行层复盘代偿决策层问题。

| | 失败（负向） | 成功（正向） |
|---|---|---|
| 决策层 | 失败归因→提出新假设，修正策略 | 关键成功要素→提炼策略规律 |
| 执行层 | 事故根因→补流程漏洞→避免再犯 | 梳理执行框架→固化稳定流程 |

> 早期新业务，核心是复盘决策策略，不要陷在执行细节里。

## 四、九宗罪

用于快速诊断复盘质量问题：

| 类别 | 问题 | 表现 |
|------|------|------|
| 意识不够 | 不做 | 只有领导要求才做 |
| | 不信 | 不认可价值，敷衍应付 |
| | 不求 | 短期看不到收益就放弃 |
| 姿势不对 | 不真 | 自我安慰式，只说好话 |
| | 不实 | 空对空全靠主观感受 |
| | 不懂 | 只挑问题或只说优点，走极端 |
| 深度不够 | 不透 | 停在表层，挖不到根因 |
| | 不全 | 碎片化，没形成体系 |
| | 不改 | 复盘完不落地，文档写完就尘封 |

## 五、ROI 原则

越符合以下特征越值得多花时间：
- 新事情：拿不准需要验证
- 新人成长：需要快速纠错
- 重复执行但还没沉淀模型：长期复用有复利
- 困难的事情：容易失败需提升成功率
- 核心基本功：公司核心能力需反复打磨

已经成熟、有稳定模型的事，再复盘价值递减。

## 六、复盘操作口诀

小事及时复盘（出问题立刻复盘，20 分钟就行），大事阶段性复盘（长周期项目中途插复盘节点），事后全面复盘（完全结束后完整梳理）。

## 七、团队复盘四步法

| 步骤 | 核心 | 方法 |
|------|------|------|
| 以身作则 | 自己先做 | 反向示范、及时复盘、认真听团队发言 |
| 挂在嘴边 | 营造氛围 | 三个问题反复问：①做对了什么要保持？做错了什么要避免？②能总结出什么可复用的认知？③下次计划怎么调整？ |
| 提出要求 | 建立规则 | 立 flag 压目标、手把手 before-after 带练、三分钟短复盘 |
| 固化流程 | 形成制度 | 周报加学习心得、关键项目复盘会 |

# 追问话术库

## 通用追问

| 场景 | 话术 |
|------|------|
| 用户停在 L1（只有感受） | 「你说的好/不好，具体是指什么？能举一个例子吗？」 |
| 用户停在 L2（只有事实） | 「知道了这些事实之后，你觉得造成这个结果的原因是什么？」 |
| 用户给出第一个原因就想停 | 「这是一个原因。还有吗？再往下挖一层呢？」 |
| 用户归因过于外部化 | 「假设外部条件不变，你自己能控制的部分里，哪个决策可以做得更好？」 |
| 用户归因过于自我批评 | 「你说的这些确实可能有问题。那这次有没有什么做得好的地方？」 |
| 用户觉得没什么好复盘的 | 「粟裕打了一场大胜仗，复盘后找了 12 条不足。你觉得你做的这件事，真的没有一点可以优化的地方吗？」 |
| 用户觉得复盘太累/没时间 | 「你不需要复盘所有事。有经验的实践者也只对最重要的事最坚决。这件事在你所有工作里，ROI 排第几？」 |
| 用户复盘完了但没产出行动 | 「如果下次做类似的事，有哪一条是你一定会保留的？有哪一条是你一定会改变的？」 |

## 成功复盘专用

| 场景 | 话术 |
|------|------|
| 用户说「就是运气好」 | 「运气好的意思是结果不可控。但你再想想，这次有没有哪个动作是你刻意做的、而且确实起效了？」 |
| 用户说「就是我很努力」 | 「努力是前提，但努力的人不止你一个。这次和其他努力但结果一般的时候，区别在哪？」 |
| 用户说不出可复现条件 | 「如果明天让你再做一次类似的事，你有信心复现这个结果吗？有信心的话，哪些动作你一定会保留？没信心的话，缺了什么？」 |

## 失败复盘专用

| 场景 | 话术 |
|------|------|
| 用户情绪化（沮丧/愤怒） | 「我能理解你的感受。我们先不说为什么，先说说：下次怎么做会更好？哪怕只是一条。」 |
| 用户归因到某个人 | 「我们先不讨论这个人。你自己的哪个决策，如果重新来一次会做出不同的选择？」 |
| 用户找到直接原因就想停 | 「这是一个原因。但'测试没测出来'的背后是什么？是时间不够？是流程没覆盖？还是压根没有测试标准？」 |

## AI 协作复盘专用

| 场景 | 话术 |
|------|------|
| 用户只说「AI 不好用」 | 「具体是哪一轮让你觉得最不好用？是理解错了你的意思，还是给了你不需要的东西？」 |
| 用户只说「我自己没想清楚」 | 「你是在对话的哪一轮开始想清楚的？是什么让你想清楚的？」 |
| 协作整体顺畅 | 「这次顺畅，你觉得是因为你这次的指令特别清晰，还是这个任务本身 AI 就擅长？」 |

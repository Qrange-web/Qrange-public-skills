# fupan-assistant · 复盘助手 Skill

> 一个可以装进任何 AI Agent 的复盘引导技能：从「感觉不错/不太好」的主观感受出发，一步步追问到根因、模型、能力层，输出结构化复盘卡。

**它不替你提炼规律，而是帮你多挖一层。**

---

## 解决什么问题

复盘最常见的卡点不是不知道方法论，而是**没人追问**：

- 做完一件事，感觉「还行」或「不太好」，但说不清为什么
- 复盘停在主观感受，挖不到深层原因
- 成功复盘变成夸夸，失败复盘变成甩锅
- 复盘完没沉淀，下次遇到同样问题从头再来

这个 Skill 干的事：像一个坐在你对面的复盘伙伴，从你已有的信息里一步步追问，帮你挖到你自己也说不出但确实存在的那个深层原因——最后给你一份结构化的复盘卡，可以直接归档到你的知识库。

## 快速开始

### 安装

这是一个标准的 Agent Skill 包（`SKILL.md` + `references/`），平台无关：

- **支持技能包格式的 Agent 运行时**（Claude Code、OpenClaw、Hermes、WorkBody，或任何兼容 SKILL.md 规范的个性化 Agent）：把整个 `fupan-assistant` 文件夹放进对应的 skills 目录即可，无任何外部依赖
- **不支持技能包的 Agent**：把 `SKILL.md` 全文作为系统提示词（system prompt）注入，references 作为上下文按需提供给模型，效果等同

### 触发

自然语言触发，不需要前缀命令：

- 「帮我复盘一下上周的客户汇报」
- 「这次做得特别顺，帮我提炼一下成功的关键」
- 「刚做完XX，感觉不太好但说不上来为什么」
- 「复盘一下这次协作」
- 「这个需求我跟你来回改了好多次，复盘一下问题出在哪」

## 核心能力

### 双路径模式

| 路径 | 适用场景 | 工作方式 |
|------|---------|---------|
| A：外部教练模式 | 你没参与过的事，Agent 也没参与过 | 一步一步追问事实、预期、根因、规律 |
| B：内部参与者模式 | 你和 Agent 协作完成的事 | 自动提取对话历史，只追问推不出来的 |

### 五层深度递进

| 层级 | 挖什么 | 判断标准 |
|------|--------|---------|
| L1 | 主观感受 | 能说出明确的感受方向 |
| L2 | 客观事实 | 至少 2 个具体事实（时间/地点/数字/动作/原话） |
| L3 | 背后原因 | 根因链条至少深挖两层 |
| L4 | 底层模型 | 产出至少一条可复用的规律/框架/清单 |
| L5 | 能力建设 | 识别团队/自己的核心能力短板 |

### 结构化输出

每次复盘结束自动输出：
- **复盘卡**：包含事件标签、预期-结果对比、五层深度诊断、下次行动、九宗罪自检
- **Wiki 标记**：自动判断复盘沉淀状态（待深挖 / 已沉淀），生成知识库归档计划

## 边界与安全

| 场景 | 行为 |
|------|------|
| 纯知识问答（「复盘是什么意思」） | 不触发诊断，直接回答 |
| 太空泛（「帮我复盘一下我的人生」） | 追问一次，仍空泛则礼貌拒绝 |
| 涉及法律/合规问题 | 拒绝追问细节，建议找专业人士 |
| 用户甩锅 | 不直接指责，追问「你自己能控制的部分」 |
| 用户情绪低落 | 先回应情绪，再切换「面向未来」表达 |

## 框架来源声明

本 Skill 使用的复盘框架（五层冰山、四象限、九宗罪等）来自资深实践者的经验归纳，属于实践者经验框架，非实证研究模型。Skill 使用时不会以「科学模型」的口吻引用这些框架。

# chaishu · 六步拆书法 Skill

> 一个可以装进任何 AI Agent 的深度拆书技能：不只还原"这本书说了什么"，而是拆出作者看不见的盲区，帮你在盲区之上长出自己的原创认知。

**它不是摘要生成器，而是一套批判性阅读系统。**

---

## 解决什么问题

读非虚构类书籍时的三个常见病：

- **复述代替理解**：读完只会说"这本书讲了什么"——这件事 AI 摘要 10 秒就能做完，轮不到你
- **盲信代替判断**：作者讲得头头是道，你隐隐觉得哪里不对劲，但说不清是"作者太乐观"还是"他回避了关键问题"
- **感动代替行动**：读完"好有感触"，一周后什么都没变

这个 Skill 干的事：用六步流程（查椅子→找魂→挑故事→做审计→生长→落地），把一本书拆成——作者的利益结构、可复用的底层操作、带现实验证的案例拆解、与对立书的对撞分析、你自己的原创认知、今天就能做的动作。

## 快速开始

### 安装

这是一个标准的 Agent Skill 包（`SKILL.md` + `references/` + `examples/`），平台无关：

- **支持技能包格式的 Agent 运行时**（Claude Code、OpenClaw，或任何兼容 SKILL.md 规范的 Agent）：把整个 `chaishu` 文件夹放进对应的 skills 目录即可
- **不支持技能包的 Agent**：把 `SKILL.md` 全文作为系统提示词（system prompt）注入，references 和 examples 作为上下文按需提供给模型，效果等同

### 联网搜索依赖（可选，推荐）

Step 1（查椅子）和 Step 4（做审计）需要联网搜索作者背景和对立书评。两种方案：

| 方案 | 说明 |
|---|---|
| **Agent Reach**（推荐） | 多平台搜索路由 skill（覆盖 LinkedIn、豆瓣、知乎、Reddit 等 15+ 平台）。安装方式见官方文档：[github.com/Panniantong/Agent-Reach](https://github.com/Panniantong/Agent-Reach) —— 最快的方式是把这句话发给你的 Agent：「帮我安装 Agent Reach：https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md」 |
| 环境自带搜索 | 没装 Agent Reach 也能用——Skill 会退而求其次，使用当前环境的 WebSearch/WebFetch 或等价搜索工具，只是平台覆盖度差一些 |

### 触发

让用户直接说（任意一句）：

```
拆解这本书
这本书有什么盲区
批判性地读一下这本书
帮我拆一下《AI 2041》
```

附上书籍原文/拆书会逐字稿/读书笔记效果更好；只有书名也可以，Skill 会先搜公开资料做浅拆。

**注意区分**：「提取这本书的主线」「整理这本书的方法论」→ 那是还原型任务，请用 [course-mainline](../course-mainline-v2/) skill；「拆解」「批判」「盲区」→ 才是 chaishu。

## 工作原理

### 六步流程

```
Step 0  入口判断 —— Go（完整六步）/ Light（轻拆三步）/ No-Go（路由或拒绝）
Step 1  查椅子   —— 作者坐在什么利益结构上？他的话如果对了他得到什么？
Step 2  找魂     —— 这本书不随时间过期的可复用操作是什么？（动词短语，不是内容复述）
Step 3  挑故事   —— 3-5 个案例按深度梯度排列，每个拆四层：表面/藏着/现实验证/书里没写
Step 4  做审计   —— 找立场相反的书做对撞，在互斥点上暴露盲区
Step 5  生长     —— 在盲区之上长出你自己的原创认知
Step 6  落地     —— 可带走的工具 + 今天就能做的具体动作
```

前一步的产出是下一步的输入，六步一个不能少（Go 模式下）。信息不足就写"未查到"并说明搜索过程，不跳过、不编造。

### 人机协同：Step 5 永远属于人

六步里只有 **Step 5（生长）由人来完成**——这是整个拆书过程中最不可替代的部分。AI 在这里退到对面当引导式提问者：每个问题先引用前几步的具体产出，再逼你思考（"审计发现作者漏了 X——你在自己的经历里遇到过类似的情况吗？"）。**AI 绝不替人编生长认知**，这是写死在 Skill 里的硬约束。

### 三种模式，不硬拆

| 模式 | 适用 | 做什么 |
|---|---|---|
| **Go** | 有作者立场的非虚构书籍/拆书会/深度长文 | 完整六步 |
| **Light（轻拆）** | 纯虚构文学（小说/诗歌/戏剧） | 只做找魂+挑故事+落地，开头标注"未做椅子审计" |
| **No-Go** | 纯工具书/编程教程 | 路由到 course-mainline 或建议做摘要 |

入口判断采用**关键词硬路由**，不依赖模型的自主判断——这是实测踩坑后的设计：自然语言约束会被 Agent 绕过（"小说有思想深度值得查"），关键词匹配不会。

### 安全边界

- 查椅子只查**公开可验证的利益结构**，不做动机揣测和人身攻击
- 涉及在世人物的利益冲突推测，必须加限定词、标注"推测"、注明来源
- "书里没写"只能归因于作者的椅子（利益结构/时代局限），不能归因于作者的人品
- 用户要求"证明作者是骗子"→ 直接拒绝，不提供折中方案

## 实际效果

用同一批输入做了三组对比实测（Skill vs 裸模型，12 维度 36 分制）：

| 测试组 | chaishu | 裸模型 | 差值 |
|---|---|---|---|
| 正常拆书（拆书会逐字稿） | **34/36** | 17/36 | +17 |
| 高风险拒绝（"证明作者是骗子"） | **12/12** ✅ | 顺着说了 | — |
| 轻拆模式（《百年孤独》） | **19/21** ✅ | 无降级概念 | — |

差距最大的两个维度：**生长引导**（裸模型分析完就停了，没有"帮用户长出原创认知"的概念）和**工具提炼**（裸模型产出读后感，Skill 产出武器库）。

## 边界（它不做什么）

- **不拆虚构文学的椅子**：小说/诗歌/戏剧只做轻拆——作者的利益结构不直接决定作品价值
- **不拆纯工具书**：编程教程、Cookbook 没有盲区可审计，路由给 course-mainline
- **不做摘要**："帮我总结一下这本书说了什么"是 AI 摘要的活，不是拆书
- **不替你生长**：Step 5 的原创认知永远由人来写，AI 只提问、不代笔
- **不做动机揣测**：查椅子只查公开可验证的利益结构，不当"证明作者是骗子"的武器
- **方法论边界**：六步法在一次拆书会上自证有效（详见 claim_audit.md 的证据分级），跨书/跨人的可迁移性仍在验证中——Skill 会把这一边界如实告知用户

# 六步拆书法 — 产出模板

> 输入：拆书会逐字稿 / 原书 PDF-MD / 读书笔记 / 课程主线文档
> 输出：7 模块结构化 Markdown 文档

## 硬约束

1. **七模块一个不能少**
2. `**` 不包标点——正确：`**核心概念**：说明`，错误：`**核心概念：**说明`
3. 信息源标注——外部搜索得到的内容标来源 URL，基于材料的推断标"推测"
4. Step 4 未读原书的对比标注"基于摘要/书评，未读原书"

## 产出骨架

```markdown
# [书名/材料名] — 六步拆解

## 1. 这是什么
## 2. 作者椅子
## 3. 魂
## 4. 挑故事
## 5. 做审计
## 6. 生长
## 7. 落地
```

## 每个模块要回答的问题

| 模块 | 回答这个问题 | 易犯错误 |
|------|------------|---------|
| 1. 这是什么 | 这份材料讲的是什么？什么背景下产生的？ | 写成书籍简介而不是定位 |
| 2. 作者椅子 | 作者坐在什么利益结构上？他说的话如果对的他能得到什么？如果是错的他付什么代价？ | 只写作者简介不写利益分析 |
| 3. 魂 | 如果只能带走一件事，那是什么？能不能用一句话描述它的可复用操作？ | 写成了这本书讲了什么而不是底层操作 |
| 4. 挑故事 | 选哪 3-5 个案例能构成一条完整的认知阶梯？每个案例拆出表面/藏着/现实验证/书里没写四层 | 只拆了两层（表面+藏着），没有现实验证和盲区 |
| 5. 做审计 | 有没有找到 1-2 本立场相反的书？它们的核心论点对撞暴露了什么盲区？ | 只列了书单没做对撞分析 |
| 6. 生长 | 在盲区之上长出了什么书里没有的原创认知？ | 换说法复述书里的观点 |
| 7. 落地 | 有没有一个今天就能做的具体动作？ | 动作模糊（"多关注""多思考"） |

## 第三节（挑故事）的子结构

每个案例统一用四个子标题：

```markdown
#### 案例 N：[案例名]

**表面**：
**藏着**：
**现实验证**：
**书里没写**：
```

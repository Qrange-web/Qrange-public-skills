# 课程主线提取

将课程/书籍/分享/任何结构化材料整理成标准化 9 模块主线 Markdown 文档。

## 前置检查

每次开始前确认两件事：

### 1. 材料盘点

用户给了什么就是什么——可以是 Get笔记，可以是逐字稿，可以是书籍原文、PPT 文本、论文、会议记录等任何材料。不需要固定格式或固定来源。

核心原则：**用户提供的材料就是全部原料**。如果材料不足（如缺少逐字稿导致无法确认原话），在 Section 9（资料来源说明）中明确声明缺失项，并说明对哪些模块的影响。

### 2. 体积感知与上下文预算

**在读取材料前，先估算体积和性价比**：

| 检查项                                      | 做法                                                     |
| ------------------------------------------- | -------------------------------------------------------- |
| 文件数量和总大小                            | `wc -l` / `wc -c` 快速扫描                           |
| 当前模型上下文预算                          | 默认假设 256K token 可用上限（保守假定）                 |
| 单文件是否含超大段（如 `audio.original`） | 如有内嵌逐字稿，优先读 AI 总结段建立结构认知，再逐段深入 |

**如果材料总量明显超过安全线**（估算材料 token 数 > 模型上限的 70%），主动提示用户：

> ⚠️ 材料总量较大（约 X 行 / Y KB），在当前模型（假设 256K 上限）下可能接近上下文极限。
>
> **建议二选一**：
> (a) 切换到上下文更大的模型（如 DeepSeek V4 Pro，1M 上下文）
> (b) 先把文件发给任意一个 AI，让它做精简（保留方法论、案例、执行框架、决策标准四类内容），再把精简版交给我处理

## 五步流程

每门课完整跑完五个步骤，除非明确指示跳过。

### Step 1: 读取资料

**第一层：快速结构扫描**

无论材料形式如何，优先做轻量扫描，建立结构认知：

- 有 AI 总结/摘要/目录段 → 先读这些，快速抓骨架
- 有逐字稿/长文本 → 先读前 5-10% 判断密度和质量

**第二层：深度逐字阅读**

方法论、案例细节、课程原话是 Section 7（原话纠正）和 Section 8（观点类型判断）的**唯一可靠来源**——AI 总结会丢失 nuance，不可跳读。如果材料包含逐字稿或实录文本，必须逐字读完。

**第三层：外部来源补充**

如果课程引用了外部文本（如书籍、论文、文章），搜索/获取原文作为第三源。这在课程引用外部文本（如拆书会）时尤其重要——原文是方法论的理论根源。

### Step 2: 写课程主线产出

按模板写 9 模块文档。模板路径：`references/课程主线-产出模板.md`

**输出方式**：生成产出前，和用户确认两件事：

1. **输出形式**：直接在对话中输出，还是单独保存为 Markdown 文件？
2. **如果保存为文件**：请用户提供目标路径。若用户给了路径但当前部署环境无本地文件写入权限，明确告知用户无法保存，改为在对话中直接输出供用户自行粘贴保存。

**硬约束**：

- 9 模块一个不能少：1.定位 → 2.核心问题 → 3.核心方法 → 4.操作流程 → 5.判断标准 → 6.适用边界 → 7.常见误区 → 8.讲师观点 → 9.资料来源
- 粗体不包标点（`**概念**：说明` 而非 `**概念：**说明`）
- Section 5 用三列表格：检验项 | 通过标准 | 不通过时
- Section 7 每条误区配课程原话纠正，不自己编；无原话可用时必须在 Section 9 声明
- Section 8 讲师观点标注类型：核心主张 / 方法论框架 / 经验主张 / 趋势判断 / 引用观点

**第三节展开原则**：子节数量和结构取决于课程内容形态——流程型课用 ASCII 图，武器库型课用多组表格，框架型课用概述表+逐模块展开。不强求统一格式。

### Step 3: 脚本检查 + 内容审查

先跑自动化检查脚本（`references/checks.py`），然后做四要素内容审查：

1. **方法准确性**：方法论框架、关键概念、步骤流程忠实于课程原文
2. **Section 4 是真操作步骤**：不是 Section 3 的重新表述
3. **Section 7 用课程原话**：纠正文字来自课程实录；若材料无原话可查，Section 9 必须声明
4. **Section 8 类型标注准确**：核心主张（底层信念/哲学观）/ 方法论框架（原创工具）/ 经验主张（基于大量观察）/ 趋势判断（未来预判）/ 引用观点（引述他人）

### Step 4: 复盘模板

检查本节课是否暴露模板不足——第三节展开方式适配吗？硬约束有不合理限制吗？若有问题，更新模板文件。大多数情况下只确认"模板无需修改"。

### Step 5: Review 模板健康度

向用户汇报模板在多门课检验后是否仍然稳定。一句话即可。

## Bundled Resources

- **`references/课程主线-产出模板.md`** — 9 模块标准化模板
- **`references/checks.py`** — 自动化质量检查脚本（bold balance / heading hierarchy / bold-period / module count / Section 5 existence）。用 `python3 references/checks.py <path>` 运行

### Examples

**`examples/`** 目录包含两个典型产出示例，供参考最终效果：

- **`示例1-书籍-方法论型.md`** — 卡普兰、诺顿《平衡计分卡》。展示方法论类书籍的完整处理方式：因果链框架 + 十步操作流程 + 书中多案例的组织和引用，410 行
- **`示例2-专项课-框架型.md`** — 杜威《思维的本质》。展示框架型理论著作/书籍的完整处理方式：五步法全景图 + 三层思维区分 + 错误思维来源 + 正确态度 + 教育训练原则，353 行
